# Implement FastDDS Backend for pj_bridge

## Context

The project has two DDS backends: ROS2 (`ros2/`) and RTI Connext (`rti/`). We want to add a third backend using eProsima Fast DDS 3.4.0 (acquired via Conan), and stop compiling the RTI backend (without deleting it). FastDDS produces OMG IDL schemas and CDR-encoded messages, same as RTI — so the core `app/` layer and PlotJuggler client need no changes.

## Architecture: Flattened (2 classes + main)

Unlike RTI's 4-class two-level design, we flatten into 2 classes that directly implement the abstract interfaces:

- **`FastDdsTopicSource`** — implements `TopicSourceInterface`, manages `DomainParticipant`s, discovers topics via `on_data_writer_discovery()`, resolves `DynamicType` from `TypeObjectRegistry`, generates IDL via `idl_serialize()`
- **`FastDdsSubscriptionManager`** — implements `SubscriptionManagerInterface`, creates `DataReader`s with `DynamicPubSubType`, ref-counted subscriptions, extracts CDR via deserialize+re-serialize
- **`main.cpp`** — CLI11 parsing + `std::chrono` event loop (same pattern as RTI)

## Step 2: FastDdsTopicSource

Implements `TopicSourceInterface`. Merges RTI's `DdsTopicDiscovery` + `RtiTopicSource` into one class.

**Reference files**: `rti/src/dds_topic_discovery.cpp`, `rti/src/rti_topic_source.cpp`

### Constructor: `FastDdsTopicSource(const std::vector<int32_t>& domain_ids)`

For each domain:
1. Create `DomainParticipant` via `DomainParticipantFactory::get_instance()->create_participant(domain_id, qos)` with TypeLookup enabled:
   ```cpp
   DomainParticipantQos pqos;
   pqos.wire_protocol().builtin.typelookup_config.use_client = true;
   pqos.wire_protocol().builtin.typelookup_config.use_server = true;
   ```
2. Attach a `ParticipantListener` (inner class) overriding `on_data_writer_discovery()`

### ParticipantListener::on_data_writer_discovery()

When `reason == DISCOVERED_WRITER`:
1. Get topic name from discovery info
2. Get `TypeObject` from `ITypeObjectRegistry` using type identifier from `info`
3. Build `DynamicType`: `DynamicTypeBuilderFactory::get_instance()->create_type_w_type_object(type_object)->build()`
4. Check `dynamic_type->get_kind() == TK_STRUCTURE`
5. Generate IDL: `idl_serialize(dynamic_type, oss)`
6. Cache in `topics_` map (deduplicated by topic name)

### Interface methods

- `get_topics()` → snapshot `topics_` under shared lock, map to `TopicInfo`
- `get_schema(name)` → return cached IDL string
- `schema_encoding()` → return `kSchemaEncodingOmgIdl`

### Non-interface methods (used by subscription manager)

- `get_dynamic_type(name)` → return `DynamicType::_ref_type`
- `get_participant(domain_id)` → return `DomainParticipant*`
- `get_domain_id(name)` → return `optional<int32_t>`

### RTI → FastDDS API mapping (discovery)

| RTI | FastDDS 3.4 |
|-----|-------------|
| `DomainParticipant(domain_id)` | `Factory->create_participant(domain_id, qos)` |
| `NoOpDataReaderListener<PublicationBuiltinTopicData>` | `DomainParticipantListener::on_data_writer_discovery()` |
| `sample.data().extensions().type()` | `TypeObjectRegistry` + `DynamicTypeBuilderFactory::create_type_w_type_object()` |
| `rti::core::xtypes::to_string(type, idl)` | `idl_serialize(dynamic_type, ostream)` |
| `participant.close()` | `Factory->delete_participant(p)` |

### Thread safety: `std::shared_mutex` (write in discovery callback, read in getters)

## Step 3: FastDdsSubscriptionManager

Implements `SubscriptionManagerInterface`. Merges RTI's `DdsSubscriptionManager` + `RtiSubscriptionManager`.

**Reference files**: `rti/src/dds_subscription_manager.cpp`, `rti/src/rti_subscription_manager.cpp`

### Constructor: `FastDdsSubscriptionManager(FastDdsTopicSource& topic_source)`

### subscribe(topic_name, topic_type)

1. If already subscribed → increment ref count, return true
2. Get `DynamicType` from `topic_source_.get_dynamic_type(topic_name)`
3. Get `DomainParticipant*` and domain_id from topic_source
4. Register type: `TypeSupport ts(new DynamicPubSubType(dynamic_type)); ts.register_type(participant)`
5. Create Topic, Subscriber, DataReader with listener
6. Store in `subscriptions_` with ref count = 1

### InternalReaderListener::on_data_available(DataReader* reader)

CDR extraction via deserialize + re-serialize (cache `DynamicPubSubType` per subscription):
```cpp
DynamicData::_ref_type data = DynamicDataFactory::get_instance()->create_data(dynamic_type_);
SampleInfo info;
while (reader->take_next_sample(&data, &info) == RETCODE_OK) {
  if (!info.valid_data) continue;
  SerializedPayload_t payload;
  pub_sub_type_.serialize(&data, &payload);
  auto cdr = std::make_shared<std::vector<std::byte>>(payload.length);
  std::memcpy(cdr->data(), payload.data, payload.length);
  uint64_t ts_ns = uint64_t(info.source_timestamp.seconds()) * 1'000'000'000ULL
                 + info.source_timestamp.nanosec();
  callback_(topic_name_, std::move(cdr), ts_ns);
}
```

### unsubscribe / unsubscribe_all

Same ref-count pattern as RTI. Close DDS entities outside lock to avoid deadlock.
