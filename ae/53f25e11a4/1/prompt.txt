## Remaining Bug: Cross-Workflow Race

Both the AppImage `release` job and the Debian `release` job use this pattern:

```bash
gh release view "$tag" >/dev/null 2>&1 || \
  gh release create "$tag" --title "$tag" --generate-notes
gh release upload "$tag" ... --clobber
```

If both jobs check simultaneously, both find "not found", both try `gh release create`, one fails тЖТ `set -e` kills the step. This is unlikely (AppImage builds are slower) but not impossible, and it would cause the same intermittent failure we're trying to fix.

**Fix**: Add `|| true` after the create command:
```bash
gh release view "$tag" >/dev/null 2>&1 || \
  gh release create "$tag" --title "$tag" --generate-notes || true
gh release upload "$tag" ... --clobber
```

If `create` fails because another workflow already created it, execution continues. If it fails for a real reason, the subsequent `upload` will fail with a clear error.
