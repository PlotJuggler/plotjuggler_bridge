# Plan: Add Conda Package Build & Upload to Release CI

## Context

Users currently install pj_bridge via Debian packages or AppImage. To support installation via `pixi global install`, we need to build conda packages and publish them to a prefix.dev channel. Each ROS distro (humble, jazzy, kilted) gets its own package (`pj-bridge-ros2-humble`, `pj-bridge-ros2-jazzy`, `pj-bridge-ros2-kilted`) since they link against different ROS2 libraries.

## Prerequisites (manual, one-time)

1. Create a channel on https://prefix.dev (e.g. `plotjuggler` or `pj-bridge`)
2. Go to channel → "Trusted Publishers" tab
3. Add trusted publisher:
   - GitHub org: `PlotJuggler`
   - Repository: `plotjuggler_bridge`
   - Workflow filename: `conda.yaml`
4. Click Submit

## Step 1: Create `conda.recipe/recipe.yaml`

This is a Jinja-templated recipe. The `ros_distro` variable is passed at build time via `--variant`.

```yaml
context:
  version: "0.4.0"
  ros_distro: humble  # overridden by --variant at build time

package:
  name: pj-bridge-ros2-${{ ros_distro }}
  version: ${{ version }}

source:
  path: ..
  use_gitignore: true

build:
  number: 0
  script: |
    cmake $CMAKE_ARGS \
      -GNinja \
      -DCMAKE_BUILD_TYPE=Release \
      -DCMAKE_INSTALL_PREFIX=$PREFIX \
      -S . -B build
    cmake --build build --target install

requirements:
  build:
    - ${{ compiler('c') }}
    - ${{ compiler('cxx') }}
    - cmake >=3.14
    - ninja
    - pkg-config
  host:
    - ros-${{ ros_distro }}-rclcpp
    - ros-${{ ros_distro }}-ament-cmake
    - ros-${{ ros_distro }}-ament-index-cpp
    - ros-${{ ros_distro }}-sensor-msgs
    - ros-${{ ros_distro }}-nav-msgs
    - ixwebsocket
    - spdlog
    - zstd
    - nlohmann_json
  run:
    - ros-${{ ros_distro }}-rclcpp
    - ros-${{ ros_distro }}-ament-index-cpp
    - ixwebsocket
    - spdlog
    - zstd

tests:
  - package_contents:
      lib:
        - lib/pj_bridge/pj_bridge_ros2

about:
  homepage: https://github.com/PlotJuggler/plotjuggler_bridge
  license: AGPL-3.0-only
  license_file: LICENSE
  summary: Bridge server forwarding ROS2 topic content over WebSocket to PlotJuggler
```

**Key decisions:**
- `source.path: ..` — recipe is in `conda.recipe/`, source root is parent
- `$CMAKE_ARGS` — auto-populated by rattler-build with cross-compilation flags
- ROS deps use `ros-${{ ros_distro }}-*` naming (robostack convention)
- The `ament_cmake` build system handles install targets automatically (binary goes to `$PREFIX/lib/pj_bridge/pj_bridge_ros2`)

## Step 2: Create `.github/workflows/conda.yaml`

GitHub Actions matrix with one job per ROS distro. Uses `prefix-dev/rattler-build-action` to build, then uploads to prefix.dev via OIDC trusted publishing.

```yaml
name: Conda Packages

on:
  push:
    tags: ['*']
  workflow_dispatch:

permissions:
  contents: read
  id-token: write  # Required for OIDC trusted publishing

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  build:
    strategy:
      fail-fast: false
      matrix:
        include:
          - ros_distro: humble
            robostack_channel: robostack-humble
          - ros_distro: jazzy
            robostack_channel: robostack-jazzy
          - ros_distro: kilted
            robostack_channel: robostack-kilted
    runs-on: ubuntu-latest
    name: conda-${{ matrix.ros_distro }}

    steps:
      - uses: actions/checkout@v4

      - name: Build conda package
        uses: prefix-dev/rattler-build-action@v0.2.36
        with:
          recipe-path: conda.recipe/recipe.yaml
          artifact-name: conda-${{ matrix.ros_distro }}
          build-args: >
            --variant ros_distro=${{ matrix.ros_distro }}
            -c ${{ matrix.robostack_channel }}
            -c conda-forge

  publish:
    name: Upload to prefix.dev
    needs: build
    if: startsWith(github.ref, 'refs/tags/')
    runs-on: ubuntu-latest

    steps:
      - uses: prefix-dev/rattler-build-action@v0.2.36
        with:
          setup-only: true

      - uses: actions/download-artifact@v4
        with:
          path: artifacts
          pattern: conda-*
          merge-multiple: false

      - name: Upload to prefix.dev
        shell: bash
        run: |
          shopt -s nullglob globstar
          EXIT_CODE=0
          for pkg in artifacts/**/*.conda artifacts/**/*.tar.bz2; do
            echo "Uploading ${pkg}"
            if ! rattler-build upload prefix -c <CHANNEL_NAME> "${pkg}"; then
              EXIT_CODE=1
            fi
          done
          exit $EXIT_CODE
```

**Note:** Replace `<CHANNEL_NAME>` with the actual prefix.dev channel name after creating it.

## Step 3: Verify the CMake install targets work standalone

The current `CMakeLists.txt` installs:
- `pj_bridge_ros2` → `lib/pj_bridge/pj_bridge_ros2` (line 158-160)
- `pj_bridge_app` headers → `include/` (line 111 area)

The recipe uses `cmake --build build --target install` which will use these targets. The `ament_cmake` build type should work since robostack provides it. If `ament_cmake` is not found in the conda env (e.g. if rattler-build doesn't source the ROS setup), we may need to set `AMENT_PREFIX_PATH=$PREFIX` in the build script. This is a known pattern with robostack — will verify during testing.
